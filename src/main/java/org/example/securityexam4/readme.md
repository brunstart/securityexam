개발순서

목적 - 시큐리티를 이용해서 인증 인가를 테스트 해보는데, 우리 DB에 있는 사용자를 기준으로 테스트

그러려면 : 회원이 필요

1. 회원가입
    - 회원가입 폼
    - 엔티티 -- 1   O
    - 레파지토리 -- 2   O
    - 서비스 - 회원가입 메소드, 회원가입 로직    O
    - DTO O
    - 컨트롤러 - 회원가입 폼, 회원가입    O


2. 로그인
    - 로그인 폼 O
    - 로그인 DTO -- 생성해서 할건지 파라미터로 직접 받을지 결정 
    - 컨트롤러 - 로그인 폼을 보여달라는 요청, 로그인 요청(필요?)
    - UserDetailService -- 구현체를 만들어 줘야함 O
    - 사용자 권한에 따라 다르게 볼 수 있는 페이지 생성
